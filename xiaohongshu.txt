🐝 开源了！让AI像麦肯锡团队一样协作 | Agent Swarm

正文：

做了半个春节的 AI 蜂巢项目项目，终于开源了！🎉

先说痛点👇

用过 AutoGen、CrewAI 这些多 Agent 框架的朋友应该都有感觉：
❌ 角色要自己预定义，换个任务就得重写
❌ Agent 各干各的，最后结果像拼凑的
❌ 产出看着花哨，但没法直接用

说白了，现在的多 Agent 系统就像一条固定流水线🏭
——你让它做流水线上的事没问题，换个任务就傻了

---

所以我做了 Agent Swarm 🐝

核心思路：不要流水线，要项目制团队

就像麦肯锡接到一个新项目，不会派固定的几个人去做，而是根据项目需要，从人才库里挑出最合适的专家，临时组队、协作、交付，做完解散。

Agent Swarm 就是这个逻辑👇

💡 四个核心能力：

1️⃣ 动态角色涌现
你说"分析《花样年华》的镜头语言"，系统自动涌现出：
🎬 镜头分析师
📖 叙事结构师
🎭 视觉符号学家
🎵 配乐解读者

不是简单取个名字！每个角色都有完整的工作目标、方法论、成功标准和预期交付物。换个任务？全部重新涌现，永远是最合适的团队。

2️⃣ 中继站实时对齐
别的框架：Agent 各干各的，最后把结果堆一起
Agent Swarm：有个"战情室"，Agent 发现关键信息会实时广播，遇到不确定会求证，到了关键节点会同步

效果就是——产出是交叉验证过的，不是拼凑的！

3️⃣ 专业技能注入
纯靠 LLM 推理，输出不稳定、能力有天花板
Agent Swarm 双通道注入：
📝 系统提示 → 植入专业方法论（让思维专业）
🔧 工具能力 → 赋予搜索/分析等执行力（突破纯推理上限）

4️⃣ 交付物导向
不是让 AI 在那废话连篇
每个 Agent 涌现时就锚定了"你必须交付什么"
有目标、有方法、有标准、有产出
出来的是能直接用的东西，不是空洞的"建议"

---

🛠️ 技术栈：
后端：Python + FastAPI
前端：React + TypeScript
协议：AG-UI（实时流式）
支持：OpenAI / Claude 等主流模型

⚡ 三种启动方式：一键脚本 / Docker / 手动分别启动
小白也能 5 分钟跑起来

---

📌 适合什么场景？
✅ 深度内容分析（电影/文学/行业研究）
✅ 创意策划（广告/品牌/营销方案）
✅ 复杂问题拆解（需要多视角协作的任务）
✅ 任何"一个人搞不定，需要一个团队"的事
