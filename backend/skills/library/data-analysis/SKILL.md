# Data Analysis

分析数据集、识别模式、生成洞察并创建可视化。适用于数据探索、统计分析或数据驱动决策。

## Workflow

1. **理解数据**: 了解数据背景和目标
   - 明确分析目的和问题
   - 了解数据来源和含义
   - 确定关键指标和维度

2. **数据探索**: 初步了解数据特征
   - 查看数据结构和类型
   - 计算描述性统计量
   - 识别缺失值和异常值

3. **数据清洗**: 准备高质量数据
   - 处理缺失值
   - 修正数据错误
   - 统一数据格式

4. **深度分析**: 挖掘数据规律
   - 进行统计分析
   - 识别相关性和趋势
   - 发现异常和规律

5. **可视化呈现**: 用图表展示发现
   - 选择合适的图表类型
   - 设计清晰的可视化
   - 突出关键发现

6. **洞察总结**: 提炼分析结论
   - 总结主要发现
   - 提出可行建议
   - 指出数据局限性

## Analysis Techniques

### 描述性分析

| 统计量 | 用途 | 示例 |
|--------|------|------|
| 均值/中位数 | 集中趋势 | 平均用户年龄 |
| 标准差 | 离散程度 | 收入分布范围 |
| 分位数 | 数据分布 | 用户消费层次 |
| 相关系数 | 变量关系 | 浏览量与转化率 |

### 可视化选择

```
数据类型 → 图表推荐

趋势变化 → 折线图
类别比较 → 柱状图
占比分布 → 饼图/环形图
相关关系 → 散点图
数据分布 → 直方图/箱线图
多维数据 → 热力图/雷达图
```

## Guidelines

- 始终考虑数据的统计显著性
- 区分相关性和因果性
- 注意样本偏差和选择偏差
- 可视化应简洁清晰
- 结论要有数据支撑

## Examples

```
需求: 分析网站用户行为数据

分析过程:
1. 数据概览
   - 数据量: 10万条访问记录，30天
   - 关键指标: PV、UV、跳出率、转化率

2. 主要发现
   发现1: 移动端占比持续上升
   - 数据: 移动端 UV 占比从 45% 升至 62%
   - 图表: [趋势折线图]
   - 建议: 优先优化移动端体验

   发现2: 下午 3-5 点是转化高峰
   - 数据: 该时段转化率比平均高 35%
   - 图表: [时段分布柱状图]
   - 建议: 在此时段增加营销投放

3. 结论
   - 需加大移动端投入
   - 营销预算应向下午时段倾斜
```

## Safety Checks

- 验证数据质量和可靠性
- 注意个人隐私数据保护
- 对于敏感结论需多次验证
- 标注分析的置信度和局限性

## Success Criteria

- 分析过程清晰、可复现
- 发现有数据支撑、有价值
- 可视化清晰、易于理解
- 结论具有可操作性
- 局限性和假设透明说明
